## 二、测试用例

### 2.1 已合并分支提醒功能

#### TC-001: 新创建的分支不应显示"已合并"

| 项目 | 内容 |
|------|------|
| **前置条件** | 存在一个 Git 仓库，有 main 分支 |
| **测试步骤** | 1. 基于 main 创建新分支 feature/test-new<br>2. 不做任何提交<br>3. 查看应用中的分支状态 |
| **预期结果** | 新分支 **不应** 显示"已合并"提示 |
| **优先级** | P0 |

#### TC-002: 有未提交改动时不应显示"已合并"

| 项目 | 内容 |
|------|------|
| **前置条件** | 存在已合并到 main 的分支 feature/merged |
| **测试步骤** | 1. 切换到 feature/merged<br>2. 修改某个文件但不提交<br>3. 查看应用中的分支状态 |
| **预期结果** | 该分支 **不应** 显示"已合并"提示 |
| **优先级** | P0 |

#### TC-003: 真正已合并的分支应正确显示提醒

| 项目 | 内容 |
|------|------|
| **前置条件** | 存在分支 feature/completed，已合并到 main |
| **测试步骤** | 1. 确保 feature/completed 已合并<br>2. 工作区干净（无未提交改动）<br>3. 查看应用中的分支状态 |
| **预期结果** | 显示"已合并"提示，提示可以删除 |
| **优先级** | P0 |

### 2.2 Pull 功能

#### TC-004: Pull 按钮显示条件

| 项目 | 内容 |
|------|------|
| **前置条件** | 存在有远程跟踪的分支 |
| **测试步骤** | 1. 查看有远程分支的 worktree<br>2. 查看没有远程分支的 worktree |
| **预期结果** | 有远程分支时显示 Pull 按钮（同步状态 ahead/behind）<br>无远程分支时不显示 |
| **优先级** | P1 |

#### TC-005: Pull 操作成功

| 项目 | 内容 |
|------|------|
| **前置条件** | 远程分支有新的提交，本地 behind |
| **测试步骤** | 1. 点击 Pull 按钮<br>2. 观察操作结果 |
| **预期结果** | 显示成功消息，分支状态更新，behind 数量归零 |
| **优先级** | P0 |

#### TC-006: Pull 冲突处理

| 项目 | 内容 |
|------|------|
| **前置条件** | 本地和远程有冲突的修改 |
| **测试步骤** | 1. 点击 Pull 按钮<br>2. 观察冲突提示 |
| **预期结果** | 显示"Pull 产生冲突，请手动解决"的提示 |
| **优先级** | P1 |
