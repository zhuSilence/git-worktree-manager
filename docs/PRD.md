## 一、产品概述

### 1.1 产品背景

Git worktree 是 Git 2.5+ 引入的功能，允许同一仓库拥有多个工作目录，非常适合并行开发场景。然而，目前市面上缺乏**免费、轻量、跨平台**的可视化管理工具：

| 工具 | 问题 |
|------|------|
| GitKraken | 付费，且功能臃肿 |
| GitLens | 依赖 VS Code |
| Git Extensions | 仅 Windows，界面老旧 |
| 命令行 | 学习成本高，无可视化 |

### 1.2 产品定位

**轻量级、免费、跨平台的 Git Worktree 可视化管理器**

- 一句话描述：让 Git worktree 像切蛋糕一样简单
- 目标用户：开发者（尤其是需要频繁切换分支的程序员）
- 核心价值：提升并行开发效率，降低 worktree 使用门槛

### 1.3 目标用户

| 用户画像 | 场景 | 痛点 |
|----------|------|------|
| **全栈开发者** | 同时开发多个功能分支 | 切换分支需要 stash，麻烦 |
| **团队 Tech Lead** | Code Review + 自己开发 | 需要同时看多个分支代码 |
| **开源贡献者** | 同时处理多个 PR | 每个 PR 一个 worktree |
| **运维/DevOps** | 紧急 hotfix | 快速切换，不影响当前工作 |

### 2.2 功能清单

| 功能模块 | 功能点 | 描述 |
|----------|--------|------|
| **Worktree 管理** | 列表展示 | 显示所有 worktree，分支名、路径、状态、最后提交 |

| | 删除 worktree | 删除保护 + 自动备份，有未提交更改时提示确认 |

| | 同步状态 | 领先/落后远程提交数，一键推送/拉取 |
| | 搜索排序 | 关键词搜索过滤 |

| 功能模块 | 功能点 | 描述 |

## 三、数据模型

### 3.1 Worktree 核心模型

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | Worktree 唯一标识符 |
| name | string | Worktree 名称 |
| branch | string | 所在分支名 |
| path | string | 文件系统路径 |
| status | WorktreeStatus | Clean/Dirty/Unpushed/Conflicted/Detached/Unknown |
| lastCommit | LastCommit | 最后提交信息（hash/message/author/timestamp） |
| lastActiveAt | string? | 最后活跃时间 |
| isMain | boolean | 是否为主 Worktree |
| syncStatus | SyncStatus | 远程同步状态（ahead/behind/hasRemote） |
| tags | string[]? | 标签列表 |
| notes | string? | 备注 |
| groupId | string? | 所属分组 |

### 3.2 操作日志模型

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 日志 ID |
| operationType | OperationType | Create/Delete/Switch/Prune/BatchDelete |
| timestamp | string | ISO 8601 时间戳 |
| target | string | 操作对象（分支名或路径） |
| details | string? | 操作详情 |
| result | OperationResult | Success/Failed |
| errorMessage | string? | 错误消息 |
| repoPath | string | 仓库路径 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 启动时间 | < 2 秒 |
| worktree 列表加载 | < 500ms (100 个 worktree 内) |
| 内存占用 | < 100MB (空闲状态) |
| 安装包大小 | < 50MB |

### 4.2 兼容性要求

| 维度 | 支持 |
|------|------|
| 操作系统 | macOS 10.15+, Windows 10+, Ubuntu 18.04+ |
| Git 版本 | Git 2.5+ (建议 2.17+) |
| 架构 | x64, arm64 (Apple Silicon) |

### 4.3 可用性要求

- 界面简洁直观，新用户无需教程即可上手
- 支持键盘快捷键操作
- 支持深色/浅色主题
- 中英双语支持（i18next）

### 4.4 安全性要求

- 不存储 Git 凭据
- 不上传任何用户数据
- 路径验证（validate_path）防止路径遍历
- 分支名清理（sanitize_branch_name）防止注入
- AI API Key 加密存储
